Welcome to JSLoader

This is a js-architecture. It requires jQuery and RequireJS. Advantages:

  • Abstract Code
  • All dependencies in 1 file
  • Easy usage

Structure

  • scripts/
    • app/
      • extensions/
        • lib/
      • appMain.js
      • extensionLoader.js
    • lib/
      • jquery.js
      • require.js
    • main.js

scripts/lib

Here are all third-party js-libraries.

scripts/app/extensions

You store your executables here.

scripts/app/extensions/lib

You store your own libraries here.

main.js

Here is the requireJS Configuration. It's already preconfigured for the basic functionality. Just change the baseUrl to your needs. That's it.

appMain.js

Here goes the code you always need.

extensionLoader.js

Here is where the "magic" comes from. Partially. 

extensionDependecies = {
    "#main": {  
        "extensions": ["extensions/test"],  
        "callback": function(){}  
    },
    ...
}

In the extensionDependecies-Object, just add a markup-selector on which your js-extension depends. The Markup-Selector stores an object containing an extensions-array and a callback-function. Write all extensions which are directly related to the markup-selector into the extensions array. The Callback-function will be executed after loading the extension. You can write a small callback here, but try not to overload the callback. It's not even required.

extensions/test.js

define(["jquery", "extensions/lib/Module"], function(jQuery, Module) {
    var $j = jQuery.noConflict();

    $j(document).ready(function() {
        var Demo = new Module();
        Demo.doStuff();
    });
});

This file stores the code that depends on "#main". In the first line, we define our dependencies for this file. In this case, it's jquery and extensions/lib/Module. Those files are loaded before executing the rest of this file. Thanks to requireJS.

extensions/lib/Module.js

define([], function() {
    return (function(){
        var _priv = null;

        function Module() {
        }

        Module.prototype = {
        }

        return Module;
    })();
});

This is how you would use the JS-Modulepattern together with this architecture.
